# خطوات رفع المشروع على Railway (للوصول من أي مكان)

بما أن الكود أصبح جاهزًا الآن، اتبع الخطوات التالية لربط المشروع بحسابك على Railway.

## 1. رفع الكود إلى GitHub
لقد قمت بالفعل بتجهيز الملفات (`Procfile`, `requirements.txt`, `.gitignore`) وتنظيف المشروع.
سيتم رفع التعديلات تلقائيًا عند تشغيل أمر الرفع (الذي سأقوم به أو يمكنك القيام به).

## 2. إعداد مشروع جديد على Railway
1. اذهب إلى [Railway.app](https://railway.app) وسجل الدخول بحساب GitHub الخاص بك.
2. اضغط على زر **+ New Project**.
3. اختر **Deploy from GitHub repo**.
4. اختر المستودع (Repository) الخاص بهذا المشروع (`-Guarantees`).
5. اضغط **Deploy Now**.

## 3. إضافة قاعدة البيانات (PostgreSQL)
بمجرد أن يبدأ المشروع، نحتاج لإضافة قاعدة بيانات دائمة:
1. داخل صفحة المشروع في Railway، اضغط على زر **+ New** (أو كليك يمين في المساحة الفاضية).
2. اختر **Database** -> **PostgreSQL**.
3. انتظر قليلاً حتى يتم إنشاء قاعدة البيانات.
4. سيقوم Railway تلقائيًا بربط قاعدة البيانات بالمشروع عبر متغير `DATABASE_URL`. لا تحتاج لفعل أي شيء إضافي، الكود مهيأ لقراءته.

## 4. التحقق من العمل
1. بعد انتهاء الـ Build والـ Deploy (يستغرق بضع دقائق).
2. سيظهر لك رابط (URL) للموقع الخاص بك (ينتهي بـ `up.railway.app` أو مشابه).
3. اضغط عليه ومبروك! الموقع يعمل الآن أونلاين.

## ملاحظات هامة
- **البيانات القديمة:** البيانات الموجودة على جهازك (SQLite) **لن** تنتقل تلقائيًا إلى السحابة. ستبدأ بقاعدة بيانات فارغة.
- **اسم المستخدم وكلمة المرور:** سيتم إنشاء مستخدم `admin` بكلمة مرور `admin` تلقائيًا عند أول تشغيل، أو يمكنك استخدام صفحة التسجيل إذا كانت مفعلة.
- **الأمان:** تأكد من تغيير كلمة المرور فور الدخول.
